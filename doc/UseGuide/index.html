JokerFrameWork简介

JokerFrameWork目的

JokerFrameWork是一个用Java写的攻击框架，主要想要解决以下痛点：

  1. 完成漏洞分析后没有统一的漏洞留存方式
  2. 完成利用链（反序列化利用链）的分析后，都是保存在各自的ysoserial项目中的，并不能进行统一的留存，这样就导致很多利用链基本无法完成复用。
  3. 从漏洞库转换成武器库

JokerFrameWork的依赖组成

JokerFrameWork的主要依赖为：

依赖 版本 说明
SpringBoot 1.5.8 SpringShell自带的SpringBoot版本
SpringShell 2.0.0.RELEASE 用于构建交互式命令行
jcommander 1.78 用于exp插件单独运行完成参数解析
unboundid-ldapsdk 5.0.1 用于创建Ldap Server
permit-reflect 0.3 ysoserial依赖
system-rules 1.19.0 ysoserial依赖

构建工具为Gradle。

JokerFrameWork项目组成

JokerFrameWork项目主要由三部分组成：

  • Joker：JokerFrameWork主项目，提供交互式命令行、插件加载、服务开启等服务
  • JokerSDK：用于exp插件开发的规范接口，及yso利用链集成。
  • exploitLib：exp插件开发项目，主要用于以插件的形式提供exp。
